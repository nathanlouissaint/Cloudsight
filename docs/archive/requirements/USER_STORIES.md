# CloudSight User Stories

## Authentication

### Registration

As a user, I want to create an account so that I can access my CloudSight dashboard securely.

### Login

As a user, I want to log in with my credentials so that I can access protected cost data.

### Protected Access

As a user, I want dashboard routes protected so that unauthenticated users cannot view cost information.

## Dashboard

### Summary Metrics

As a user, I want to view current month spend, previous month spend, forecasted spend, and budget usage so that I can quickly understand cloud cost status.

### Budget Risk

As a user, I want to see whether my current spend is within budget so that I can respond before overspending.

## Cost Analysis

### Daily Trends

As a user, I want to view daily spending trends so that I can identify cost spikes.

### Monthly Trends

As a user, I want to compare monthly spending so that I can understand long-term cost movement.

### Service Breakdown

As a user, I want to see spending by cloud service so that I can identify which services drive cost.

### Top Cost Drivers

As a user, I want to see the highest-cost services so that I can prioritize optimization work.

## Budget Monitoring

### Create Budget

As a user, I want to create a monthly budget so that CloudSight can compare actual spend against my target.

### Budget Status

As a user, I want to see budget status as healthy, warning, or exceeded so that I can quickly understand risk.

## Forecasting

### Moving Average Forecast

As a user, I want the system to forecast future spend using recent spending patterns so that I can estimate month-end cost.

### End-of-Month Projection

As a user, I want projected month-end spend so that I can compare expected spend against budget.

## Anomaly Detection

### Spend Spike Detection

As a user, I want the system to detect sudden spending increases so that I can investigate unusual cloud activity.

### Budget Breach Detection

As a user, I want the system to flag budget breaches so that I can take corrective action.

## Executive Reporting

### Monthly Summary

As an executive stakeholder, I want a plain-English monthly summary so that I can understand cloud spend without reading raw billing data.

### Optimization Suggestions

As a user, I want basic optimization suggestions so that I know where to investigate potential savings.

## Portfolio Review

### Interview Explanation

As a job candidate, I want this project to demonstrate cloud architecture, AWS concepts, full-stack development, DevOps, and documentation so that I can discuss it in interviews.
